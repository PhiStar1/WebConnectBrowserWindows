WebConnect Features für zukünftige WebConnect-Updates
In Arbeit:
- Ein Bug wird behoben, der dazu führt, dass wenn man aus einer anderen App einen Link öffnet, ein zusätzlicher Tab mit der vom Benutzer eingestellten Startseite geöffnet wird.
Ausstehend:
- Tabs speichern (nach Start soll gefragt werden ob die Tabs wiederhergestellt werden sollen)
- Perplexity AI, YouChat,  als unterstützter Chatbot
- Favoritenleiste
- Smart Reload von Webseiten für Entwickler (Webseite wird automatisch neu geladen sobald Änderungen an der Datei sind)
- Überarbeitete MDM Settings
- Man soll Cookies und Browserdaten löschen können
- Wenn man einen Tab rauszieht soll der in einen neuen fenster sein und man soll den auch wieder in andere WebConnect-Fenster reinziehen können
- Automatisches herunterladen von Softwareupdates
- Translations mit Sprachdateien
- Chronik
- Plugins inkl. Plugin-IDE
- Benutzersystem mit normalen Usern (inkl. User-Name und optionales Anmeldekennwort und folgenden Extra-Nutzern: Admin, Gast. Für den Admin-Benutzer ist ein Kennwort sehr empfohlen. Gast surft im Inkognito modus)
- Bootloader (bootloader.exe), Recovery (recovery.exe), Anmeldefenster (wclogon.exe) und eigentlicher Browser (WebConnect.exe + dessen Dateien) in seperaten Dateien
  - Bootloader prüft sich selbst, Hauptanwendung + Anmeldeprozess vor dem Start auf fehler und bei Bedarf auf Malware.
  - Wird ein Fehler/Malware gefunden, wird eine Fehlermeldung angezeigt und das Recovery mode wird gestartet.
  !!Bei Malwareprüfung beim Start kann der Startvorgang von WebConnect um einige Zeit dauern!!
- Wechsel zwischen Dark- und White Mode
- Modernere maybe runde Buttons
- Animationen von Buttons, Textboxen und anderen Elementen, mit denen der Nutzer interagieren kann
- Tab-Ordner
- Hotkey-Editor
- Webseiten-Blacklist in den Mobile Device Management Einstellungen
- Ghost Tabs (unsichtbare Tabs)

- Umstieg auf Chromium?
